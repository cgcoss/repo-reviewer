# AGENTS.md — repo-reviewer

Wails v2 desktop app: Go backend + React/TypeScript/Tailwind frontend. A read-only Git working-tree changes viewer.

## Prerequisites

- **asdf** with `.tool-versions`: Go 1.26.2, Node.js 22.14.0
- **Wails CLI v2**: `go install github.com/wailsapp/wails/v2/cmd/wails@latest`

## Commands

| Task | Command |
|------|---------|
| Install deps | `asdf install && cd frontend && npm install` |
| Dev (hot reload) | `wails dev` |
| Build release | `wails build` |
| Run Go tests | `go test ./internal/git/...` |

- `wails dev` launches the desktop app **and** the Vite dev server. Do not run `npm run dev` directly.
- `wails build` on macOS produces `build/bin/repo-reviewer.app`.

## Architecture

- **Backend entry**: `main.go` → `app.go`
- **Go packages**:
  - `internal/git/` — git CLI wrapper (status, diff, branch). Tests live here.
  - `internal/watcher/` — `fsnotify` watcher with 200 ms debounce; emits Wails runtime event `git:status-changed` on repo changes.
- **Frontend entry**: `frontend/src/main.tsx` → `App.tsx`
- **Frontend bindings** (`frontend/wailsjs/go/main/`) are **auto-generated by Wails**. Do not hand-edit.
- **Assets**: `//go:embed all:frontend/dist` in `main.go` — the Go binary embeds the built frontend. `frontend/dist` is in `.gitignore`.

## Important Conventions

- No frontend test suite exists; only Go backend tests.
- `go.mod` pins `go 1.23.0` as the module version, but the project is developed with Go 1.26.2 (see `.tool-versions`).
- The watcher watches the working tree recursively (skipping `.git/`) plus specific `.git/` subpaths (`index`, `HEAD`, `refs`).
- Untracked file diffs use `git diff --no-index` against `/dev/null` (or `NUL` on Windows).
- Status parsing expects porcelain v1 with `-z` (`\x00` separators).
