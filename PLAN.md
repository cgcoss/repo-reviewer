# repo-reviewer Implementation Plan

## Overview

A lightweight Wails + Go desktop app that opens a local Git repository and displays its working tree changes in an IntelliJ-style Git Changes panel.

## Prerequisites (must install first)

1. Install Go 1.26.2 via asdf
2. Install Wails CLI: `go install github.com/wailsapp/wails/v2/cmd/wails@latest`
3. Set Node.js version: `asdf local nodejs 22.14.0`
4. Verify: `wails doctor`

## Core Feature to Implement

1. User can choose or enter a local repository path
2. Backend validates that the path is inside a Git repository
3. Backend reads Git status using native `git` commands, not a heavy Git library
4. Frontend displays changed files grouped like IntelliJ:
   - Changes
   - Staged Changes, if any
   - Untracked Files, if any
5. Each file row shows:
   - filename
   - relative path
   - Git status badge, such as M, A, D, R, ??
6. Selecting a file shows a diff preview in the main pane
7. Diff preview should support:
   - unified diff text view
   - syntax-neutral styling
   - added/removed line highlighting
8. Add a refresh button that reloads status and selected diff
9. Add basic error states:
   - invalid repo path
   - git not installed
   - no changes
   - diff command failed

## UI Direction

- IntelliJ-inspired layout:
  - left sidebar: repository path and change tree
  - main pane: diff viewer
  - bottom/status area: current branch, file count, last refresh time
- Dark theme by default
- Compact, dense UI
- Keyboard-friendly where simple:
  - up/down changes selection
  - Cmd/Ctrl+R refreshes
- Tailwind CSS for all styling

## Backend Requirements

- Expose Wails methods roughly like:
  - `OpenRepository(path string) (RepoSummary, error)`
  - `GetStatus(path string) ([]ChangedFile, error)`
  - `GetDiff(path string, file ChangedFile) (string, error)`
  - `GetCurrentBranch(path string) (string, error)`
- Use `git -C <repo> status --porcelain=v1 -z` for robust parsing
- Use `git -C <repo> diff -- <file>` for unstaged files
- Use `git -C <repo> diff --cached -- <file>` for staged files
- Handle untracked files gracefully; show a synthetic diff or file preview if possible, but it is acceptable to show "Untracked file: no diff available yet" for this slice
- Avoid shell interpolation; use `exec.Command` args safely

## Data Model

```go
type ChangedFile struct {
    Path      string `json:"path"`
    OldPath   string `json:"oldPath,omitempty"`
    FileName  string `json:"fileName"`
    Status    string `json:"status"`
    Staged    bool   `json:"staged"`
    Untracked bool   `json:"untracked"`
}
```

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| CSS framework | Tailwind CSS | User preference; fast styling, utility-first |
| Go version | 1.26.x | Latest stable as of now |
| Project name | repo-reviewer | User preference |
| UI framework | React + TypeScript (Vite) | Wails-native, type-safe, minimal |
| Git interaction | `exec.Command` with `git` CLI | Per spec; no heavy library |
| Porcelain parsing | `-z` NUL-delimited | Handles spaces and special chars in filenames |
| Untracked diff | `git diff --no-index /dev/null -- <file>` | Best-effort; fallback to placeholder message |

## Implementation Phases

### Phase 1: Environment Setup & Project Init
1. Install Go 1.26.2 via asdf
2. Install Wails CLI
3. Rename directory `version-reviewer` -> `repo-reviewer`
4. Run `wails init` with the React template
5. Replace the default frontend with Vite + React + TS + Tailwind
6. Verify `wails dev` launches successfully

### Phase 2: Go Backend
1. Create `internal/git/git.go` — `CheckGitInstalled()`, `ValidateRepo()`
2. Create `internal/git/status.go` — `ParseStatus()` with porcelain parser
3. Create `internal/git/diff.go` — `GetUnstagedDiff()`, `GetStagedDiff()`, `GetUntrackedDiff()`
4. Create `internal/git/branch.go` — `GetCurrentBranch()`
5. Create `app.go` — `OpenRepository()`, `GetStatus()`, `GetDiff()`, `GetCurrentBranch()` bound methods
6. Write `status_test.go` and `diff_test.go`
7. Run `go test ./internal/git/...`

### Phase 3: Frontend UI
1. Set up Tailwind config with Darcula-inspired dark palette
2. Create `types/index.ts` mirroring Go structs
3. Build `App.tsx` — state management, keyboard handling, layout
4. Build `RepoPathInput.tsx` — text input + Browse (Wails directory dialog)
5. Build `ChangeTree.tsx` + `FileRow.tsx` — grouped change list with status badges
6. Build `DiffViewer.tsx` + `DiffLine.tsx` — unified diff rendering with color-coded lines
7. Build `StatusBar.tsx` — branch, file count, refresh button, timestamp
8. Wire up all Wails bindings via `useWails.ts` hook

### Phase 4: Integration & Error Handling
1. Handle all error states (invalid repo, git not installed, no changes, diff failed)
2. Implement refresh button + Cmd/Ctrl+R keyboard shortcut
3. Implement arrow key navigation for file list
4. Keep untracked file diff graceful

### Phase 5: README & Polish
1. Write README with setup, dev mode, build, and limitations sections
2. Final visual polish — ensure compact density, consistent spacing, monospace diff
3. Verify `wails build` succeeds and the binary works

## Deliverables

- A working Wails app that launches
- Clear README with:
  - setup instructions
  - how to run in dev mode
  - how to build
  - current limitations
- Keep implementation scoped. Do not implement commit, push, branch management, staging, merge tools, history graph, or settings yet
- Include a few lightweight tests for the Go Git status parser

## Acceptance Criteria

- Can run the app locally
- Can point it at an existing Git repository
- Can see changed files in an IntelliJ-like changes panel
- Can click a changed file and see its diff
- Refresh updates the UI after modifying files externally
- The app remains simple and does not introduce unnecessary dependencies
